Documentação Pública - NEZA

Guia completo de funcionamento e integração da plataforma de Gestão de Riscos

1. Introdução

O Sistema Integrado de Monitoramento e Gestão de Riscos Digitais (NEZA) é uma plataforma desenhada para operações de segurança cibernética (SOC). Ele permite o registro, monitoramento e a resolução de vulnerabilidades e eventos de segurança em tempo real.

2. Como o Sistema Funciona

  • Dashboard Interativo: Exibe métricas cruciais como riscos ativos, críticos e resolvidos. O gráfico de distribuição é alimentado dinamicamente via MongoDB.
  • Gestão de Ameaças: Um módulo CRUD onde operadores podem registrar novas ameaças classificando-as por nível de severidade (Baixo, Médio, Alto).
  • Sistema de Alertas: Sempre que uma ameaça de nível Alto é registrada, o sistema dispara notificações push (Toasts) na interface.
  • Relatórios: Um log histórico de todos os eventos registrados no banco de dados para fins de auditoria.

3. API Pública (Endpoints REST)

A plataforma NEZA opera através de um backend Serverless. Se você deseja integrar o NEZA a outras ferramentas (como SIEMs, firewalls ou scripts Python), utilize os seguintes endpoints:

Listar Riscos (GET)

GET /api/risks

Retorna um array JSON contendo todas as ameaças armazenadas no banco de dados MongoDB.

Registrar Novo Risco (POST)

POST /api/risks

Espera um corpo JSON com a estrutura: { name, description, level, status }. Retorna 201 Created.

Resolver ou Atualizar (PUT)

PUT /api/risks

Espera o ID do risco e os campos a serem atualizados. Exemplo: { id: "RSK-101", status: "resolvido" }.

Remover Registro (DELETE)

DELETE /api/risks?id=RSK-XYZ

Apaga permanentemente o registro de ameaça da base de dados.

4. Segurança e Privacidade

Todos os dados transitam por HTTPS e são armazenados em um cluster seguro do MongoDB Atlas. Nenhuma credencial de API ou banco de dados é exposta no lado do cliente. Esta documentação serve como referência de integração.
